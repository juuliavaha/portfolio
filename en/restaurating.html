Restaurating

Restaurant Review App


In this project, I developed a mobile restaurant review application using React Native. The application allows users to save and manage restaurant reviews.

The goal of the project was to create an application that combines a local database, the use of mobile device features, and a clear and user-friendly interface. I used Figma to design the user interface.

The application uses an SQLite database, where reviews are stored locally without the need for a separate server. Reviews are divided into three categories (À la Carte, Fine Dining, and Street Food), and the user interface dynamically filters the reviews based on the selected category.

Adding a new review is implemented through a separate form screen, which is opened using React Navigation's Stack Navigator. In the form, the user can enter the restaurant name, comment, star rating, location, and an image. Required fields are validated before the data is saved to the database.


The user can either take a photo with the device's camera or select one from the photo library.

Location, camera, and photo library permissions are requested from the user when these features are used. The restaurant name, category, and star rating are required fields, while the other information is optional. If any required information is missing, an error message is displayed.


Technologies Used

  • React Native
  • TypeScript
  • Expo
  • SQLite
  • React Navigation (Stack Navigator & Bottom Tabs)
  • Expo Location
  • Expo Image Picker
  • Expo Camera
